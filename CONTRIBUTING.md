# Aturan Kontribusi Tim RPL-A-3

Untuk menjaga kerapian proyek **ALTIGUIDE** kita, ikuti aturan berikut yah adik-adik:

1. **Branching:** Jangan langsung push ke `main`. Gunakan branch dengan format: `fitur-[nama-fitur]` atau `docs-[nama-bab]`.
2. **Commit Message:** Gunakan awalan yang jelas, contoh:
   - `Feat: menambah fitur login`
   - `Fix: memperbaiki bug peta`
   - `Docs: update laporan bab 1`
3. **Pull Request:** Selalu buat Pull Request dan minta **Fauzil (Lead Dev)** untuk review sebelum merge.
